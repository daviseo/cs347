Absolute position


Boxes with position:static are simply laid out in the order they appear in the html and take up as much room as they need. This is also the default behavior when no css is applied to html. The major difference is wether an element is a block box, or a inline box.

Static is not the same as absolute! When someone imagines absolute they often think of it as the same as static, but they are very different. Static elements find the closest parent with non-static positioning, and the child is then bound to its parent in some way. For example, this text is positioned to fill 5 pixels from the left, bottom, and right edges of its parent, which is the text above. The code to the right describes these two blocks of text and how they are related, note the parent is relative, and the child is absolute. But wait! The code and sources to the right are also positioned absolutly to the parent text block, so that they are entirely outside of the block. 

        
        
#text {
  width: 400px;
  height: 400px;
  display: center;
  background-color: brown;
  padding: 20px;
  box-sizing: content-box;
  position: relative;
}
#p2 {
  height: 250px;
  position: absolute;
  background-color: maroon;
  padding: 10px;
  left: 5px;
  bottom: 5px;
  right: 5px;
}
